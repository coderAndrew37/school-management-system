Route & Page — app/(admin)/admin/governance/page.tsx (215 lines) is a server component that runs 8 parallel Supabase fetches and renders a 7-stat header strip (announcements, upcoming events, attendance, present/absent today, low-stock alerts, fee arrears, collected/outstanding KES) before handing off to the tabbed hub.
GovernanceHub — 5-tab client controller: Announcements · Calendar · Attendance · Inventory · Fees. Tab badges show live counts and alert dots for low stock and overdue payments.
AnnouncementsPanel — Create announcements with title, body, audience targeting (All / Parents / Teachers / Specific Grade), priority (Low → Urgent with colour coding), expiry date, and pin-to-top. List view with pin/unpin toggle and delete per row.
CalendarPanel — Full mini calendar grid with Monday-first weeks, month navigation, and per-day event dots colour-coded by category (Academic, Sports, Cultural, Holiday, Meeting). Add events with date ranges, time slots, location, and public/staff-only visibility. Upcoming events list for the next 90 days.
InventoryPanel — 10 item categories with emoji icons. Add items with SKU, unit, quantity, minimum stock threshold, condition, cost, location, and supplier. Low-stock alert banner. Search + category filter. Stock movement modal with transaction types (Received / Issued / Returned / Damaged / Disposed / Audited) and LPO reference field.
FeesPanel — Two sub-tabs: Payment Ledger and Fee Structures. Ledger lets you search students, filter by term/status, and record payments with M-Pesa code field. Fee Structures lets you define per-grade, per-term fee breakdowns (tuition + activity + lunch + transport + other). Status auto-computes via a DB trigger.
AttendanceOverviewPanel — Date picker triggers server-action refetch. Shows present/late/absent/excused counts, a stacked distribution bar, a per-grade breakdown table, and a 14-day attendance rate area chart via Recharts.
Migration 004 — Full PostgreSQL migration with all 7 tables, ENUM types, indexes, RLS policies per role, and two DB triggers (inventory quantity auto-update, payment status auto-compute).
